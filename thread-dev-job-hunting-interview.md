- ## 睡前思考了一下为什么最近招聘的前端都是做大后台的。记录一下自己的想法。
- https://twitter.com/ZeroZ_JQ/status/1762496404575162805
- 1. 最基本的市场模型可以简化为是A把产品卖给B，当然，其中涉及到很多细分的流程，但基本也都是这样一个流程。
- 2. 这个最简单的流程中涉及到两个链路： 一个是经销物流的链路，一个是传播的链路。其中传播链路又需要关心媒介和内容。
  - 广播电视时代，电视台掌握渠道，传播链路上的电视台就赚很多钱。后来进入pc时代，门户网站掌握渠道，做网站的，搞seo的，搞站群的，也都赚钱了。
  - 后来移动时代，微信，新闻app是传播渠道，头条微信开始赚钱，附带做公众号，微博账号的赚钱。
  - 现在是短视频时代，抖音是王者。
- 3. 说到底，为啥做c端的开发少了，因为需求少了，开发更多的是作为提高效率的工具来对待，已经不是掌握流量与传播渠道的，自然价值一直在衰减。
- 4. 对比几个时期的岗位，其实还是这么几个岗位，做内容的采编（小编-自媒体-短视频制作者），优化传播渠道的（seo-投流-操盘手）。
- 5. 上一代的技术会从渠道看看变成基建，上一代的产品，会慢慢变成平台。
- 6. web作为传播渠道暴力的时代已经过去，他成为了下一个时代的基建。

- 下一个传播渠道暴利的时代是啥？
  - 目前看就是短视频，走一步看一步。但是做内容永远不会错。

- ## [有知道天津云账户与自由职业者之间怎么避税合作吗？ - 知乎](https://www.zhihu.com/question/321759404)
- 灵活用工解决方案+委托代征政策，综合灵活用工的场景即临时工，兼职工等无雇佣性质的用工方式场景+税务局委托单位公司代替税务局进行代征代缴的业务，从而达到三方有利的合作关系
  - 1，企业完成合规对私结算，并且取得合规进项
  - 2，税务局征得核定个税，此前此个税是征收不上来的
  - 3，平台公司获取一定的服务费用，完成企业收入

- 正规的灵活用工平台用得好不仅可以有效降低企业用工风险、用工成本，还能解决个人开发票问题，但是水也很深，搞不好就会踩坑。
  - 灵活用工平台姑且可以把它理解为一个互联网劳务中介，企业和劳动者通过平台达成交易，平台的价值在于提供用工信息、开票结款和代征税款。但是网上的很多所谓灵活用工平台只是打着平台的幌子，实质就是个开发票的。
  - 灵活用工平台不仅要能开票，还要能为劳动者集中申报代扣代缴个人所得税，并且能够向劳动者提供个人所得税完税凭证，要做到这些就必须取得一张“委托代征税款证书”，否则就是不正规的。
- 个人的收入，不仅仅有发票增值税的问题，还有所得税的问题，增值税是在开票时就会处理完，而所得税会有一个时间周期，不是立马会显现出问题，如果平台只是开了张发票却没有为你代扣代缴个人所得税，那么日后你还是需要自己去解决。
  - 所以，当你找到一家灵活用工平台代开发票，第一件事就是问对方是否有委托代征资质，是否能提供个税完税证明，如果不能建议调头走人。
- 灵活用工平台还有一个风险点在于资金安全，因为款项都是通过平台这个中间账户来结算的，也就是说客户的款是打到平台公司账户的，然后平台公司再把款结转给你，之前也发生过平台卷款跑路的案子，所以选择灵活用工平台还是多谨慎一些吧。
  - 如果自由职业者和企业之间的交易并不是通过这个云平台接触和完成的，平台的价值只是解决发票问题，那么从一开始本质就有点变味了。
  - 自由职业者如果年收入不超过20万税负真的不高，完全可以自己去税务局申请开票，年终手机app办理汇算申报，现在税务局服务都在提升，没有大家想象的那么麻烦，这种才是最合法且经济成本低的方式。
  - 如果自由职业收入较高了，也可以办理营业执照注册个人工作室，申请核定征收方式来缴纳个人所得税，合法合规享受国家的优惠政策。
- 咱们国家对于中低收入人群本身税就不高，对于小微企业和个体工商户还有很多的优惠政策，对咱们普通人来说，依法纳税其实就是最低的成本了。

- 不是避税，是在现行法规允许下进行的节税操作。
  - 课税名目是经营所得，相关的模式也是获得所在地税务局报备后才可以开展业务的，是合规的一种降低成本方案。
  - 云账户可以理解为税务局授权的代征个税的企业，享受核定的个税税率。
  - 通过云账户等服务方来为甲方企业代发佣金，帮个人完个税，可以达到降低成本，税务合规的效果。

- 请问云账户扣税后，第二年个税可以申请退税吗
  - 云账户做的是经营所得税，不涉及到退税的事情，退的是个人劳务报酬那些

- 把综合所得转变为经营所得，可以说是换了项目课税，所以是避税还是节税，最终看税局怎么看吧。

- 就以抖音主播这类的自由职业者为例：如果结算金额超过平台规定的限额，那就不能个人账户结算了，只能选择对公账户结算。平台通常会提醒主播们签约云账户，来提升结算金额。
  - 之后平台、云账户和主播之间的合作模式如下：
  - 1、平台冻结30%样子的结算金额，剩余的70%直接打给云账户
  - 2、云账户负责开票给平台，平台收到票后再将30%的结算金额解冻
  - 3、主播的云账户个体户工作室通常按季度报税
  - 云账户需要收取一定的服务费
  - 主播的个体户工作室没有避税可能，而且无法核定征收，需按查账征收方式纳税
  - 个人经营所得税税率为5%~35%，还有增值税及附加税费

- 云账户是收取主播的服务费吗 亲知道收多少吗
  - 服务费一年三千多吧

- ## [为什么越来越多的企业会用天津云账户? - 知乎](https://www.zhihu.com/question/552070169)
- 现在社会大环境影响，很多企业招聘全职工种压力太大，有天津云账户灵活就业的劳动者出现，帮助解决事情，支付一定的劳动报酬就可以了，比长期的招聘员工来得更省心，也更节约成本
- 一是逃避社保责任；二是将不能抵扣增值税的工资、劳务费通过开具劳务费发票实现抵扣；三是共同瓜分地方税收返还
- 从我们公司与天津云账户的合作来看，真的是大大减少了用工成本。我们公司的一些器械确实需要定期维修，一次大概一周多有，如果招聘全职的维修师傅，压力会大一点，还有一些乱七八糟的事情，现在有灵活用工的维修师傅就省了很多。

- ## 💰 [请问天津云账户是做什么的公司啊? - 知乎](https://www.zhihu.com/question/552404821)
- 天津云账户是2021中国民营企业500强第243位，在天津的民营企业中给应届生开的工资算是比较高的企业之一。
  - 云账户简单来理解是做人力资源/财税领域的SaaS企业，是ToB端的业务。
  - 主要是为甲方企业合作的自由职业人员提供薪酬的结算、税款扣缴、保险代缴等服务，是自由职业者的结算平台。
- 人力资源行业粗略理解大概分为：人员招聘、人员劳动关系外包、人员管理（如中智集团、国投人力、中软外包、人瑞人才、外企德科），HR SaaS 管理软件（ADP、北森），自由职业者/兼职人群招聘（兼职猫、斗米），最后自由职业者结算的服务 就是由云账户所做的事了。
  - 主要业务内容细分的话会涉及到一些人力资源知识、财税领域的知识。
  - 因为是强政策绑定的行业，所以，需要跟政府部门打交道会比较多，故官网上会展示比较多的政策性相关文件跟新闻。

- 我们公司就有和天津云账户合作过，当时有一个项目需要一些灵活工种，就找到了天津云账户，整个合作期间领导都很满意，我也好好地交了差，下次有这样需求的话还会找他们的，专业。
  - 一样，给开发票，还在合作，平台操作也还算方便。

- ## 📌 Tips for solving leetcode codings interview problems
- https://x.com/_trish_07/status/1884688446574506073
If input array is sorted then
- Binary search
- Two pointers

If asked for all permutations/subsets then
- Backtracking

If given a tree then
- DFS
- BFS

If given a graph then
- DFS
- BFS

If given a linked list then
- Two pointers

If recursion is banned then
- Stack

If must solve in-place then
- Swap corresponding values
- Store one or more different values in the same pointer

If asked for maximum/minimum subarray/subset/options then
- Dynamic programming

If asked for top/least K items then
- Heap

If asked for common strings then
- Map
- Trie

Else
- Map/Set for O(1) time & O(n) space
- Sort input for O(nlogn) time and O(1) space

- ## [只签了3个月试用期劳动合同，公司说3个月试用期考核通过才签劳动合同，是合法的吗? - 知乎](https://www.zhihu.com/question/649996398)
- 只签3个月试用期劳动合同做法不对。
  - 试用期包含在劳动合同期限内，劳动合同仅约定试用期的，试用期不成立。
  - 如果单独签订试用期合同，就是一份无试用期的正式劳动合同，且劳动合同期限不满三个月的，不能约定试用期。
  - 试用期同样缴纳社保。社会保险即五险一金应在入职的当月（不超过30天），就要为员工参加，不能等到转正后参保。

- ## ➿ 面试官太喜欢问为什么TCP握手三次、挥手四次？
- https://twitter.com/plantegg/status/1767088277193179431
- 感觉没人说到关键点上
  1. TCP是双向的
  2. SYN的语义是建立双向连接，保证连接可靠需要3次交互
  3. FIN的语义是我方不再发送数据，但不代表对方不能继续发送，实际上是2次挥手交互
  4. 当对方也不再继续发送的时候，对方发送FIN，再产生2次挥手交互
- 握手是3次交互，单边挥手是2次交互，这么理解就简单了
  - 我方发送FIN之后，对方回ACK，此时我方处于FIN_WAIT2，对方可以选择不发送FIN，对方能继续发送业务数据，我方可以正常处理业务数据。
  - 因为TCP是双向的，SYN是建立双边连接，FIN是关闭单边连接。
  - 4次挥手实际是分开的两段式过程，可以同时也可以不同时。要把4次挥手分开成两段，才能真正理解这个过程。
- 反过来说，如果FIN的语义被定义为关闭双边连接，那么实际上也是需要3次交互，而不是4次。这就是很多人的纠结点，为什么打开是3次，关闭变4次了？
  - 关键就是，SYN是建立双边连接，FIN是通知对方我不再发送了，他们不是一回事，不是两个镜像相反的过程。
- 建立连接为什么需要3次？有人说是为了可靠性，那为什么不是100次更可靠，有人说是要取得工程上的折中所以是3次，那为什么折中点不是4次
  - TCP是双向的，收发包经过的中间链路可能并不一样，很容易出现能收不能发或者能发不能收的情况，TCP需要确认双方都相互具备收包和发包能力。
  - 相互能够确认自己和对方都同时具有收发包能力的最小交互次数就是3次。
- 那丢包怎么办？丢包通过重传来解决，但是逻辑上的交互次数是不变的，最小就是3次。3次握手交互加上重传机制，是TCP保证连接可靠的最佳工程实践。
- 既然握手的时候已经确认了连接可靠，那为什么挥手的时候还需要两次交互？我方发起FIN通知对方我方不再发送数据，对方回不回ACK又有什么关系呢，反正我方已经不发送了，这不是多此一举？
  - 还是又绕回到重传机制，重传机制要求对方回ACK才能实现重传逻辑，所以单边挥手最少需要2次。
- 📌 总结一下
  1. 握手时，逻辑上保证双方互相能收发包，最少需要3次
  2. 挥手是单边的，FIN的语义并不是SYN的反义词，他们有根本的不同，所以不能放一块来类比
  3. 要保证重传机制正常运作，单边挥手至少需要2次
  4. 双边挥手并不一定发生在同一时间，所以挥手的报文不一定能合并发送，挥手最少只能是4次

- 其实只要理解 TCP 是基于「流」和网络环境不可靠这两个原则，自然而然就会推导出三次握手和四次挥手。
