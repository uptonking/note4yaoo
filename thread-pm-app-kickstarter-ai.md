- ## 🤼🏻 憋了很久，但最终还是决定说出来的一个看法：最没有意义的AI应用就是那些”总结类产品“。
- https://twitter.com/onenewbite/status/1729704474040742236
  01.  它不能代替你阅读。总结的意义在于阅读后，而不是阅读前；
  02.  在“节省”时间的同时，它也让你离“一手信息源”越远。我们现在接触的很多已经是二手信息，这种没有阅读的“总结”是三手，四手，五手知识；通过”总结“学到的不是知识，是以为自己学到知识的幻觉(或虚荣心)。
  03.  总结的本质是对具象知识的吸收和高度抽象。总结的最大获益者是做出总结的人，而不是阅读总结的人。跳过具象知识，就像猪八戒吃人参果：”你不知道有籽没籽儿，有核没核，它到底是个什么味儿啊”-一个80后西游记迷控制不住的reference.
  04.  试图通过阅读总结学知识，是很多人”懂得很多道理，却依然过不好这一生”的原因。学知识，获得智慧，没有捷径。
  05.  一个更加现实的问题：你无法确定AI总结的是准确的。想要确定它是准确的，你需要去读一下。所以，why even bother? （如果是读书，人家已经有了一个准确的summary: 目录。如果是读论文，人家已经有了摘要。而书籍和论文是最好的信息源）
  06.  这种AI summary甚至都不是可靠的内容筛选器。即使它总结的准确，这种高度抽象会导致那些越是正确的内容，被总结出来的越是那些“老生常谈”的大道理。看起来都千篇一律。你会pass掉很多好的内容。
  07.  这个帖子仅限于“总结类”产品。而不是“解释类”产品。
- 利益相关：作为一个内容创作者，I F*king hate those AI  summary tools. 因为它们试图给观众一个幻觉：Up主花费那么长时间的努力可以被他们那些**没有任何原创性和附加值的软件**简单一两句话总结代替。所以，我你可以说我是Biased。我接受。但，你也有权利知道我的Bias。最后，欢迎各种机器人在这条推文下总结。
  - 这不是一条我认为绝对正确的推文。它是出自我利益冲突下的一条推文。所以接受一切反驳

- 脱离场景讲道理是不对的
- 没意义不代表没需求。UP主输出的已经是二手知识，但很多人认为他需要三手，四手知识。有没有意义和有没有需求不是一回事儿。但AI总结做不到“寓教于乐”。

- ## 全国信息安全标准化技术委员会《生成式人工智能服务安全基本要求》（征求意见稿）发布
- https://twitter.com/xiaohuggg/status/1712475802863034851
  - [《生成式人工智能服务 安全基本要求》（征求意见稿） + pdf](https://www.tc260.org.cn/front/postDetail.html?id=20231011143225)
  - 要求：提供者不应使用未经主管部门备案的基础模型进行研发。在训练过程中，应将生成内容安全性作为评价生成结果优劣的主要考虑指标之一。
  - 《要求》提出，建立语料来源黑名单，不得使用黑名单来源的数据进行训练。应对各来源语料进行安全评估，单一来源语料内容含违法不良信息超5%应将其加入黑名单。
  - 使用包含人脸等生物特征信息的语料时，获得对应个人信息主体的书面授权同意，或满足其他合法使用该生物特征信息的条件。

- ## 👾🔍 Perplexity 昨天刚刚宣布了新一轮 7300 万美金的 B 轮融资，正好最近自己也在做 http://devv.ai 这个产品，有很多思考，randomly 分享一下：
- https://twitter.com/Tisoga/status/1743182100541653370
- 1. 冰山理论
  - 22 年底 Perplexity 还没有那么多人知道的时候，我们就实现过一个 demo 版本，当时的想法是这难道不是另一个 GPT wrapper (现在肯定还有很多人有这种想法)，但是我们 demo 没有做到的是，如何把搜索速度做得足够快，如何把体验做得足够好。
  - 在真的做了 http://devv.ai 之后，我们才发现背后有太多事情要做了，不是简单的在 LangChain 上接个 embedding database 就能做好的，RAG 的每一个环节都有很多优化的地方要做 (可以看我之前写的两篇 RAG 的分享)。
  - Google 搜索也就是一个框，但是藏在冰山下的是巨大的复杂细节。
- 2. UX Matters
  - Perplexity 另一个优势是在 UI & UX 上使用足够简单，过去 20 年搜索引擎已经把用户养成了搜索这个行为习惯，改变习惯是最难的事情。
  - Perplexity 在保持搜索这个方式的同时做了很多 UX 的优化，这比 ChatGPT browsing mode 的体验好太多了 (当然 OpenAI 的目标是 AGI，做一个好用的产品可能只是副业)。
- 3. Think big, Start small, Iterate
  - Perplexity 的愿景是构建世界上最好的问答引擎，但是想要一步到位是不可能的，最开始的版本就是基于 GPT API + Bing API，先把这个事情做出来，然后再迭代。
  - [Think Big, Start Small, Iterate - #01 | Jiayuan Thoughts](https://jiayuan.zhubai.love/posts/2117979470253125632)
- 4. Subscription 可能不是最好的商业化模式
  - Generative Search Engine 的成本会比传统的搜索引擎成本高很多，保守估计 Perplexity 2023 年 5 亿次搜索的成本大概在几千万美金，而目前的 ARR 应该是无法 cover 这个成本的，未来可能有更多的商业化尝试的可能性 (基于 LLM 的广告平台？)

- Arvind 在discord里提过pplx目前是profitable的

- 问题是创新点就是搜索+llm，如果商业证明可行，现有的无论哪个搜索引擎抄一个，分分钟秒掉啊。我在一家大搜待过，基本上每个细分领域都有很多人在优化。我感觉有点为他人做嫁衣裳的感觉。至少在国内会是这样。

- ## 上周跑了5个城市 - 北京、上海、台北、香港以及山东某烧烤城市，办了两场派对+平均每天见5-6个创始人和投资人。聊了很多朋友，整理一下我的思考和体会：

01.     做AI要短平快
02.     做Crypto看周期
03.     做生意而不是创业
04.     做让人上瘾的产品
05.     抓住窗口期不内耗
06.     好的胃+好睡眠很重要
07.     让你的名字值钱

08.     做AI要短平快
   - 长期主义的情怀和故事是不可少的，但是抓AI的机会一定要短平快。AI的确是这几年最有爆发力的风口，但是在行业Hype顶端建立的长期主义大概率会被摔死。
   - 对普通创始人而言，不要盲目做8个月以上的AI项目，8-12个月足够技术再革新一代，先赚到钱才能踩上下一波浪，小心被巨头踩死。

09.     做Crypto看周期
  - 团队背景和技术有多牛都不如周期看得准。当下的加密圈仍然是一个充满名利欲的小赌场，行业的盘子和用户基础其实支撑不起来大项目（当然该吹吹），做项目准确看到每一波周期的热点并且在合适的时间投入才能被大家抬起来。
  - 作为创始人要坚信去中心化的理念，但是不要自己感动自己。

10. 做生意而不是创业
  - 很遗憾但是接下来2年不适合我们所习惯过去20年的创业模式，而更适合“做生意”。高利率、低经济增长、恶化的投资环境都意味着cost of capital上升，投资难拿，退出难搞。
  - 现在不是拿着c端注册量b端意向书就能一轮轮拿钱的时候了，一定要做好现金流，对公司和个人都是落袋为安。

11. 做让人上瘾的产品
  - 在经济下行的时候创业选痛点不如选痒点，千万不能碰黄赌毒但是要思考他们背后有哪些让人性上瘾的需求。
  - 中国互联网出海最好的项目几乎都是软色情相关（约会/直播/短视频）和游戏的，本质上是多巴胺而不是科技的生意。你瞧不上的网文、玻尿酸也都是让人上瘾闷声发大财生意。

12. 抓住窗口期不内耗
  - 身边赚到大钱的年轻人多是两类：真正懂行业底层规律的思考者和能抓住窗口期和核心资源的行动者。
  - 前者早早布局，后者胆大手快，而赚不到钱的则是那些看到机会苗头要多重思考讨论还迟迟不动手的“聪明人”。
  - 做事别内耗，普通人其实没那么聪明也没有信息源，先做再想其实也不错。

13. 好的胃+好睡眠很重要
  - 身边好的创始人和投资人不一定有好的家庭资源也不一定都是名校名企好背景，甚至都不一定都那么勤奋，但是他们一定一定都有好的消化系统和睡眠系统，遇到事情都能肝上。
  - 务必把自己身体搞好，不然来了机会你也抓不住，抓住机会也赚不到钱，赚到钱也不能享受好。

14. 让你的名字值钱
  - 机会迅速变化行业人才迅速流动，个人品牌的重要性越来越高，一定要想清楚自己要做哪种生意什么样的个人品牌最能帮到你。
  - 做2C搞流量做社群也是做个人品牌，做2B/2G/投资能把核心关系点打通留下好名声也是个人品牌，但是让名字值钱可以让你后期融资、拿单、搞合作都轻松很多。
