  - [我转行程序员的经历](https://github.com/yihong0618/gitblog/issues/186)
  - 掌控感，这个月第一次经历了两个同时增长的项目，几乎所有业余时间都在这上，在这个阶段懂了，如何在自己能力范围内更新项目，判断需求，处理 issue 和如何决定是否合 PR, 当然在累的什么都不想干的时候，忽然想明白了休息才是最重要的
  - Make it work then make it better. 在最开始的时候我总是想把项目尽可能做的好，加足够多帮到大家的功能再发布，后来明白了，这个是开源项目，我需要做的是，尽可能的让项目 work 起来，pull the trigger, 如果这个项目 work 的足够好，会吸引更多厉害的人来一起 make it better
  - 别炫技，如果这个不是需要炫技的项目，那么，尽可能用最简单的实现，项目就会帮到更多的人，让更多的人参与进来。

- [马桶里的大厂病 - hayami's blog](https://hayami.typlog.io/bullshitjobs)
  - 大家知道，在大公司做亿级 dau 的产品，一个功能上线，会有测试-验收-发版-灰度-回测-ab测试-全量的步骤。
  - 回测如果发现bug, 需要 debug, 规定时间搞不定或是客户端bug，就需要回滚。
  - 为什么明明可以直接去找房东，我却花了3个小时打各种电话碰各种壁，才在最后不得已的时候求助房东呢？明明他之前也帮我修过家具呀，明明这也算是房东的某种义务。
  - 不知道大家还记不记得，我之前在 blog 里有写到，北京和上海哪个城市的心理咨询更有市场？答案是上海。因为在北京，江湖义气、称兄道弟，遇到什么事情朋友聚一聚喝个酒撸个串就过去了。但是上海更有边界感，并且更商业化，所以更倾向于用心理咨询的方式来解决情绪问题。
  - 「松弛」是如此不堪一击。
  - 用无穷无尽的加班来喂养它，用人为制造的压力来鞭打它，用不可一世的规则来异化它，绩效、考核、环评、年终都是凶手的绳索，死死勒在它脖子上。
  - 我觉得其实也不是大厂，从小到大的教育都告诉我们要努力！要向前！要加油！可是谁教过我们怎么玩耍、怎么休息、怎么爱呢？这好像是每个东亚人都需要共同面对的课题。

- [三年前端面试经验加感悟](https://juejin.cn/post/6844904089902383112)
  - 我经历的面试，很多情况下就是让你给出解决方案，考得就是技术的实际运用能力，
    - 几乎每一轮面试我都会碰到这种题目，这就要求你在平时工作中要有意识的积累，光搬砖你是说不出来完整的解决方案的
  - 不会涉及到具体的问题点，考察的就是候选人的知识广度和深度，平时的积累，考虑问题的维度等方面
    - 如果我是候选人，我可能会从长列表的优化、跨域/跨页面通信、配置化（可维护性）、数据预加载、数据缓存、页面/组件无缝切换/转场动画、离线包/ssr等角度来回答这个问题
  - 技术发展快，这条路不进则退，只有持续学习才能在技术这条路上越走越远
  - 能够把自己的技能展现出来也是一种本事，在面试的时候尤为重要，头疼

- ## 在严重内卷的大环境里，如何尽量延长程序员的职业生涯，而不被卷掉？
- https://twitter.com/idatte2018/status/1520188455694716928
  - 不能只看当下的薪酬水平、以及老板是否赏识、同事间关系是否融洽等边缘指标，
  - 而要看自己的市场价值，如果让离开当下这家公司，你还能找到什么价位的工作，千万不能本末倒置，市场价值才是核心指标。工资高而市场价值低则不可久干。
